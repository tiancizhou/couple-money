# 记录重新描述修改

## 背景
AI 解析记账时，因描述不准确可能导致分类等字段错误，用户需要对已记录的账目进行修改。

## 方案
用户点击历史记录条目，弹出编辑弹窗，可重新输入描述，AI 重新解析后直接更新原记录（保留原日期）。

## 交互流程
1. 用户在历史记录列表中点击某条记录
2. 弹出 Vant Dialog 弹窗，显示当前记录信息（金额、分类、角色、备注）
3. 弹窗内有输入框，用户输入新的描述文字
4. 点击提交 → 调用 `/api/parse` 让 AI 重新解析
5. 解析成功后调用 `PUT /api/records/:id` 更新原记录（保留原日期不变）
6. 关闭弹窗，刷新历史记录和看板数据

## 改动清单

### 后端
- `database.js`: 新增 `updateRecord(id, { amount, category, role, type, note })` 函数
- `server.js`: 新增 `PUT /api/records/:id` 路由，接收解析结果按 id 更新

### 前端
- `HistoryList.vue`: 点击记录弹出编辑弹窗，支持重新输入描述、调用 AI 解析并更新
